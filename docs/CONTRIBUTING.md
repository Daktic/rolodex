# Contributing to Rolodex

Thank you for your interest in contributing to Rolodex. We're building in public to demonstrate transparency and our technical approach.

---

## Before You Start

**Open an issue first.** For anything non-trivial, create an issue before writing code. We want alignment before effort.

- Discuss your proposed changes
- Get feedback on approach
- Avoid wasted work on features that won't be merged

---

## Contribution Workflow

1. **Fork** the repository
2. **Create a branch** from `main` for your work
3. **Make your changes** following our guidelines below
4. **Submit a PR** against `main` with a clear description

---

## Pull Request Guidelines

### Description Requirements

Your PR description must include:

- **What** you changed
- **Why** you made the change
- How to test it (if applicable)

### Keep It Concise

**Important:** With AI, code generation is now easier than code review.

- Write clear, understandable code
- Keep PRs focused and small
- More information ≠ better PR
- Succinct explanations > verbose documentation

We prioritize reviewability over completeness. A small, well-explained PR is better than a large, comprehensive one.

---

## Code Standards

- **TypeScript** - All code should be typed
- **React Native conventions** - Follow standard RN patterns
- **No unnecessary dependencies** - Question every new package
- **Local-first** - No server calls, no analytics, no tracking

---

## Contribution Ideas

We're currently focused on Phase 1 (core exchange functionality), but we welcome contributions in these areas:

- Bug fixes and performance improvements
- Protocol specification refinements
- Documentation and code clarity
- Developer experience improvements
- Compatible tooling

**Have a bigger idea?** Open an issue to discuss! We're open to proposals that align with our principles (privacy, local-first, verifiable).

---

## Questions?

Open an issue and tag it as a question. We'll respond when we can.
